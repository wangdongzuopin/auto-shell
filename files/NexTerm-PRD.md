## 1. 产品概述

### 1.1 产品名称

**NexTerm**（Next + Terminal）

下一代 Windows 本地终端工具。AI Native，不是 AI 附加。

### 1.2 产品愿景

现有终端工具（Windows Terminal、Tabby、WezTerm）都是把 AI 当插件塞进去——弹出侧边栏，让你复制粘贴去问。NexTerm 的目标是反过来：**让 AI 长在终端里**，报错时自动解释，输命令时主动补全，不需要切换任何窗口。

同时做到界面极简、配置零门槛、Windows 深度优化——这三点加上 AI，是现有所有竞品都没有同时做到的组合。

### 1.3 核心差异化

| 维度 | 竞品现状 | NexTerm |
|---|---|---|
| AI 集成 | 侧边栏插件，体验割裂 | 内嵌每个交互节点，原生感 |
| 配置门槛 | JSON / Lua / 多层设置页 | 图形界面，所见即所得 |
| Windows 适配 | 跨平台工具，Windows 是二等公民 | Windows 优先，深度系统集成 |
| 界面风格 | 功能堆砌，视觉嘈杂 | 极简留白，内容优先 |

### 1.4 目标用户

| 用户类型 | 描述 | 核心痛点 |
|---|---|---|
| 开发者 | 日常使用命令行的程序员 | 报错要切浏览器查，命令记不住 |
| 运维人员 | 管理脚本和本地服务的技术人员 | 重复命令多，缺少快捷操作 |
| 技术爱好者 | 对终端外观有审美要求的用户 | 现有工具要么丑，要么难配置 |

## 2. 核心功能

> 原则：每个功能解决一个真实痛点，做到极致。不求大而全。

### 2.1 本地终端

支持 Windows 全部主流 Shell，体验与原生一致，外观大幅改善。

**支持的 Shell：**
- Command Prompt (cmd.exe)
- Windows PowerShell 5.1
- PowerShell 7+（检测到时自动识别）
- WSL / WSL2（检测到时自动识别）
- Git Bash（检测到时自动识别）

**基本能力：**
- 完整 ANSI 转义码解析与渲染
- 上下键翻阅命令历史，Ctrl+R 模糊搜索历史
- 标准快捷键：Ctrl+C 中断、Ctrl+L 清屏、Ctrl+A/E 行首尾
- 鼠标滚轮翻看历史输出，URL 自动识别可点击

### 2.2 多标签页

- 顶部标签栏，新建 / 关闭 / 重命名（双击标签名）
- 快捷键：Ctrl+T 新建，Ctrl+W 关闭，Ctrl+Tab 切换
- 每个标签页独立进程，互不影响

### 2.3 主题与外观

**可配置项：**
- 背景色 / 前景色 / 16 色 ANSI 调色板
- 字体族、字号、行距
- 光标形状（块状 / 下划线 / 竖线）与颜色
- 背景透明度（0%～100%）

**内置主题（6 套）：** One Dark、Dracula、Nord、Monokai、Solarized Dark、Tokyo Night

主题以 JSON 格式存储，支持导入 / 导出，改动实时预览。

### 2.4 快捷命令面板

- Ctrl+Shift+P 呼出，侧边栏形式
- 保存常用命令，支持命名与分组（Git、服务、系统等）
- 点击一键发送到当前终端

### 2.5 Windows 深度集成

这是 NexTerm 区别于所有跨平台终端的核心优势：

- **右键菜单集成**：文件夹右键"在 NexTerm 中打开"，自动 `cd` 到当前目录
- **系统深色模式跟随**：自动检测 Windows 深色 / 浅色模式并切换主题
- **任务栏跳转列表**：右键任务栏图标直接新建指定 Shell 的标签页
- **原生通知**：长时间运行的命令完成后，发送 Windows 系统通知

## 3. AI 功能（核心差异化）

> AI 不是插件，是终端的一部分。所有 AI 交互都在终端界面内就地完成，不弹窗，不切换。

### 3.1 报错智能解析卡片

**触发条件**：命令执行后，退出码非 0（即报错）

**行为**：错误输出下方自动展开一张折叠卡片，包含：
- 错误原因（1-2 句话，人话解释）
- 修复建议（1-3 条，按可能性排序）
- 一键执行按钮（点击直接把修复命令写入输入框）

**示例**：
```
$ npm install
npm ERR! code EACCES
npm ERR! permission denied

┌─ AI 解析 ────────────────────────────────────┐
│ 权限不足，npm 无法写入全局目录。              │
│                                               │
│ 建议修复：                                    │
│ ① 以管理员身份运行终端          [执行]       │
│ ② sudo npm install              [执行]       │
│ ③ 修改 npm 全局目录权限         [执行]       │
└───────────────────────────────────────────────┘
```

**可配置**：自动展开 / 需点击展开 / 关闭此功能

### 3.2 自然语言转命令

**触发方式**：在输入框输入 `#` 后跟自然语言描述

**行为**：
1. 实时调用 AI 生成对应的 Shell 命令
2. 命令显示在输入框，可编辑后再执行
3. 命令旁附带简短说明（可关闭）

**示例**：
```
# 把当前目录所有 jpg 文件重命名加上今天日期
→ for %f in (*.jpg) do ren "%f" "%~nf_2026-03-27%~xf"
```

**支持的描述类型**：文件操作、进程管理、网络查询、Git 操作、服务控制等

### 3.3 命令解释悬浮卡

**触发方式**：选中终端内任意命令文本，出现悬浮工具条

**行为**：点击"解释"按钮，展开 tooltip 说明：
- 命令的整体作用
- 每个参数 / flag 的含义
- 常见用法变体

**示例**：
```
选中: grep -rn "TODO" ./src --include="*.ts"

解释卡：
grep     在文件中搜索文本
  -r     递归搜索子目录
  -n     显示匹配行的行号
  "TODO" 搜索关键词
  ./src  搜索目录
  --include="*.ts"  只搜索 .ts 文件
```

### 3.4 上下文感知补全

**与普通 Tab 补全的区别**：普通补全只基于文件系统，NexTerm 的补全结合：
- 当前目录结构
- 最近使用的命令历史
- 项目类型识别（检测到 package.json 就补全 npm scripts，检测到 Makefile 就补全 make targets）
- AI 根据上下文预测最可能的下一步命令

**触发方式**：Tab 键（与普通补全共存，AI 建议显示在普通补全下方，灰色文字）

### 3.5 AI 设置

**模型接入（可切换）：**

| 模式 | 说明 | 适用场景 |
|---|---|---|
| 云端 API | 调用 Claude / OpenAI，需填写 API Key | 效果最好，需联网 |
| 本地模型 | 调用 Ollama（本地运行的开源模型） | 离线使用，数据不出本机 |

**设置项：**
- 选择模型提供商（Claude、OpenAI、Ollama）
- 填写 API Key（加密存储在本地）
- Ollama 本地地址配置（默认 `http://localhost:11434`）
- AI 功能开关（可单独关闭每个 AI 功能）
- 自动触发 / 手动触发（报错卡片是否自动展开）

## 4. 不做的功能（v1.0）

- SSH 远程连接
- SFTP 文件传输
- 插件系统
- 分屏布局
- 云端配置同步

## 5. 非功能需求

### 5.1 性能

- 启动时间 < 2 秒（不含 AI 模型加载）
- 标签页切换响应 < 100ms
- AI 报错卡片响应 < 2 秒（云端），< 5 秒（本地模型）
- 支持同时开启 10 个标签页不卡顿

### 5.2 隐私与安全

- API Key 使用系统级加密存储（Windows Credential Manager）
- 本地模式下，命令内容完全不离开本机
- 云端模式下，仅发送必要上下文（命令 + 错误输出），不发送历史记录

### 5.3 兼容性

- Windows 10 1903+ / Windows 11，x64 优先
- 安装包体积 < 150MB

## 6. 用户界面规范

### 6.1 整体风格

参考 claude.ai：大量留白、无边框感、内容优先。AI 功能的 UI 元素克制、不打扰，用折叠卡片而非弹窗。

### 6.2 界面结构

```
┌──────────────────────────────────────────────────────┐
│  [tab 1]  [tab 2]  [+]            [AI] [cmd] [设置]  │  ← 标签栏
├──────────┬───────────────────────────────────────────┤
│          │  PowerShell 7.4    CMD    WSL              │  ← Shell 切换
│  快捷    ├───────────────────────────────────────────┤
│  命令    │                                           │
│  面板    │   终端输出区域                             │
│  (可折)  │                                           │
│          │  ┌─ AI 报错卡片（自动展开）──────────┐   │
│          │  │ 原因 / 建议 / 一键修复            │   │
│          │  └──────────────────────────────────┘   │
│          ├───────────────────────────────────────────┤
│          │  ❯  输入框（# 触发自然语言模式）          │  ← 输入区
└──────────┴───────────────────────────────────────────┘
```

### 6.3 AI 交互原则

- AI 功能默认静默，只在需要时出现（报错才展开卡片，不主动打断）
- 所有 AI 生成的内容都有明确标识（小图标 + "AI" 标签）
- 用户可以一键关闭任何 AI 功能，回到纯终端模式
