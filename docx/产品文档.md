# pizz 产品文档

> 版本：0.1.0 | 更新日期：2026-05-09

---

## 一、产品定位

**pizz** 是一款面向**独立全栈开发者**的桌面 AI 工作站。

不同于 Cursor / GitHub Copilot 等 IDE 插件模式，pizz 是一个**独立桌面应用**，核心理念是让 AI 成为"坐在你旁边的另一位开发者"，能够理解整个项目上下文、执行文件操作、并同时提供开发和产品两个视角的辅助。

**一句话定位：** 个人全栈开发者的 AI 合伙人。

## 二、目标用户

### 核心用户画像

| 维度 | 描述 |
|------|------|
| 角色 | 独立开发者 / 自由职业者 / 单人创业者 |
| 技术栈 | 全栈（前端 + 后端），常用 Vue/React + Node/Java/Go |
| 工作方式 | 独自负责从需求到上线的全流程 |
| 痛点 | 缺少代码审查、缺少产品讨论对象、重复性工作多 |
| 付费意愿 | 愿意为提升效率的工具付费（¥199-499/年可接受） |

### 次级用户

- 小型创业团队（2-5 人）
- 技术爱好者 / 学生
- 产品经理（使用 PM 角色进行需求分析）

| 知识库 | 知识条目 CRUD + 全文搜索 | 完成 |
| Diff 追踪 | AI 文件编辑记录 | 完成 |

| 对话导出 | 导出 Markdown / HTML | 完成（PM 模式） |

| MCP 支持 | Model Context Protocol 互操作 | 待开发 |

### AI 技能清单（内置 10 套）

1. 编码规范 —— 代码风格、命名、文件组织
2. 安全审查 —— OWASP Top 10、秘密检测
3. TDD 工作流 —— 测试驱动开发方法论
4. 前端模式 —— React/Vue 组件设计、状态管理
5. 架构分析 —— 系统设计、技术选型
6. PRD 撰写 —— 产品需求文档模板
7. 原型设计 —— UI/UX 原型指导
8. Mermaid 图表 —— 流程图、时序图、类图
9. 竞品分析 —— 市场调研框架
10. 代码审查 —— 质量检查清单

## 五、商业模式

### 分层策略

| 版本 | 定价 | 目标用户 |
|------|------|----------|
| 开源版 | 免费 | 个人开发者 |
| 企业版 | ¥499/年/席位 | 小型团队（2-10 人） |
| 企业定制版 | 按需报价 | 中大型企业 |

### 企业版差异化功能

- 团队协作（共享项目、共享技能）
- 云端数据同步与备份
- 使用量统计与团队管理面板
- SSO / 企业认证
- 优先技术支持
- 私有化部署

### 辅助收入

- 技能市场抽成（15-30%）
- API 代理服务（统一网关赚差价）
